*   **STRICT TABLE SYNTAX:** When outputting tables, do not append any backslashes ("\") or escape characters at the end of table lines, headers, or titles. Ensure there is a clean, empty line directly before and after every table block.

*   **ROW ORDERING RULE (TOTAL -> BREAKDOWN):** Do not separate totals and breakdowns into disconnected row groups. For each metric family, place the total row first, followed immediately by its breakdown rows in the next lines of the same table.

*   **NON-RESTRICTIVE KPI RULE:** Metric lists in this prompt are the minimum required baseline, not the full limit. *   **COLUMN-BASED OUTPUT INSTRUCTION:** Follow the exact column order provided for each section (for example: Column 1 = Metric, Column 2 = Value, Column 3 = Unit, etc.). Do not add extra columns.

*   **EMPTY CELL HANDLING:** Never leave a cell truly empty. Use "N/A" when a field is not applicable.

*   **INTERNAL KPI COVERAGE CHECK:** You must analyze every requested KPI and sub-KPI for the 2025 period, even if it is ultimately not disclosed in the report.

*   **OUTPUT FILTER RULE:** In the final output tables, include rows only for KPIs with data actually available in the report. Do not create placeholder rows for missing KPIs, and do not output "Not Found" rows in the main tables.

*   **MISSING KPI HANDLING:** If a KPI is reviewed but not disclosed, omit it from the final output tables rather than inventing, estimating, or printing a "Not Found" row.
