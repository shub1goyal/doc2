    *   Classify as **'Partial'** if the scope explicitly excludes certain entities, covers only domestic or "major" companies/operations, or covered entities/operations account for less than 90% of total revenue/operations.
    *   Classify as **'Unclear'** if the scope uses vague language that cannot be definitively verified (e.g., "covers our major plants") or if subsidiary coverage cannot be verified.
*   **GHG ORGANIZATIONAL APPROACH RULE:** If Scope 1 or Scope 2 emissions are disclosed, classifying the GHG Organizational Approach is mandatory. Select from the following hierarchy (prioritizing the first 3 control/equity approaches over the last 2):

    2. **Equity Share:** Equity stake / Net-asset value.
    3. **Operational Control:** Operating control / Operations it controls (e.g., day-to-day running, operating license).
    4. **Other Boundary Criteria Disclosed:** Disclosing specific boundaries (e.g., X amount of companies/facilities/offices/production sites) that do not fall under the first 3 control/equity definitions.
    5. **No Approach Disclosed:** Disclosing just text or tables with no supporting scope or boundary information.
    *   If the company reports multiple boundaries, choose the one matching the absolute values incorporated, in accordance with the above hierarchy (Financial Control > Equity Share > Operational Control).
*   **COMPREHENSIVE SOURCE SCANNING:** You must extract data from all content types within the document, including standard text, tables, charts, graphs, infographics, and footnotes.
*   **EXACT NUMERICAL TRANSCRIPTION:** You must transcribe all numerical values exactly as they appear in the source document. **Do not round numbers, alter decimal places, or perform any calculations.** The value you extract must be identical to the value in the report.

*   **HANDLING MULTIPLE & CONFLICTING DATA:** If you find two or more different numerical values for the exact same metric name within the document (e.g., in different tables, charts, or text), you MUST create a separate row for each distinct value. Each row must capture the specific value, unit, and page source as found in its location. Do not attempt to resolve the conflict or choose one value over the other.
*   **GRANULARITY:** PRIORITIZE GRANULARITY. Extract the most disaggregated data available. **Do not sum, average, or aggregate numbers yourself.** However, if the report provides a pre-calculated total (e.g., 'Total Scope 1 & 2 Emissions'), you must extract that value into its designated row.
*   **SOURCE TRACING:** SOURCE TRACING IS MANDATORY. In the "Page Source (PDF#)" column, list all page numbers where the data point is mentioned.
*   **NUMERICAL FORMATTING:** Present all numerical values without commas (e.g., 1234567).

*   **REVISIT ALL PAGE CONTENT:** Assume no prior knowledge. Systematically review the entire document to ensure no data is missed.

**Task 4: Handle Missing Information (Auditor Mode)**

You will now act as a strict **Data Availability Auditor**. Your sole purpose is to identify valid reasons why a specific **Environmental or Financial data point** from the tables in Task 2 or Task 3 might be missing from the report. If a KPI is missing because the company states it is "immaterial" or "non-material", include the exact qualitative justification in the Data Availability Notes. You must follow the three-step filtering process below without deviation.

**Three-Step Filtering Process:**

**Step 1: Identify Candidate Sentences**
*   Scan the document for sentences that *appear* to be about data availability, omission, or redirection to other sources.

**Step 2: Apply Mandatory Exclusion Filters**
*   For **every single candidate sentence** you identified in Step 1, you must test it against the following filters. If a sentence matches **any** of these exclusion filters, it **must be deleted** and not included in the final output.

    *   **FILTER A: Is this from the third-party Assurance/Verification Statement?**
        *   If the sentence is from the auditor's section describing the scope, limitations, or methodology of their *audit*, it is irrelevant boilerplate.
        *   **Action: DELETE IT.**

    *   **FILTER B: Is this a "Zero-Value" Finding?**
        *   If the sentence reports that there were zero incidents or no significant impact (e.g., "No environmental spills," "No significant impact on biodiversity"), it is a data point (a value of 0), not an explanation for missing data.
        *   **Action: DELETE IT.**

    *   **FILTER C: Is this about an Out-of-Scope Topic?**
        *   If the sentence refers to the absence of social or governance data (e.g., "no data breaches," "no forced recalls," "parental leave," "LTIFR," "fatalities," "political donations") or general financial/auditor metrics (e.g., "borrowings," "surplus/deficit," "sell currency," "inter-corporate deposits," "statutory dues"), it is outside the scope of our task.
        *   **Action: DELETE IT.**

**Step 3: Produce Final Output**
*   Create a final section titled "Data Availability Notes". This section will contain **only the sentences that passed all three exclusion filters** in Step 2. If no sentences pass the filters, state that "No relevant data availability notes were found."
